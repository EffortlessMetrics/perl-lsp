feat(tooling): add dead code detection automation (#284)

Implements automated dead code detection using cargo-udeps and clippy
dead_code lints to keep the codebase clean and prevent accumulation
of unused dependencies, dead code, and unused imports/variables.

## Changes

### New Files

- `scripts/dead-code-check.sh`: Main detection script with three modes
  (check, baseline, report)
- `.github/workflows/dead-code.yml`: CI workflow with three jobs
  - Full detection with baseline comparison (label-gated)
  - Fast unused dependencies check (all PRs)
  - Fast dead code lint check (all PRs)
- `.ci/dead-code-baseline.yaml`: Baseline configuration with thresholds,
  current counts, and allowed exceptions
- `docs/DEAD_CODE_DETECTION.md`: Comprehensive documentation with usage
  instructions, best practices, and troubleshooting guide

### Modified Files

- `justfile`: Added dead code detection recipes
  - `just dead-code`: Local check against baseline
  - `just dead-code-baseline`: Generate/update baseline
  - `just dead-code-report`: Generate JSON report
  - `just dead-code-strict`: Strict mode (fail on any increase)
  - `just ci-dead-code`: CI gate integration
- `clippy.toml`: Added documentation comment about dead code detection

## Features

### Tool Integration

- **cargo-udeps**: Detects unused dependencies (requires nightly)
- **Clippy lints**: Detects dead code, unused imports, unused variables
- Auto-installs cargo-udeps on first run
- Graceful degradation if tools not available

### Baseline System

- Tracks current dead code counts
- Configurable thresholds (fail if exceeded)
- Allowed exceptions for intentional unused code
- Policy configuration (enforcement level, review schedule)

### CI Integration

- **Full check** (label-gated): Runs on PRs with `ci:dead-code` label
- **Fast checks** (all PRs): Quick feedback on unused deps and dead code
- **Scheduled** (weekly): Automated monitoring on Monday 3 AM UTC
- Uploads artifacts with detailed reports
- Optionally generates PR comments with summary

### Local Development

- Simple commands: `just dead-code` for quick check
- Baseline management: `just dead-code-baseline` to update
- Strict mode: `just dead-code-strict` for pre-commit validation
- JSON reports: `just dead-code-report` for analysis

## Baseline Configuration

Default thresholds:
- Max unused dependencies: 5
- Max dead code items: 10
- Max unused imports: 20
- Max unused variables: 10

Policy:
- Enforcement: warn (non-blocking by default)
- Fail on baseline exceeded: yes
- Review interval: 30 days
